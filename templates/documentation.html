owner_admin(request)

Facilitates the administrative portal for proprietors. The function initializes by verifying the HTTP request method. Upon confirmation of a POST request, it extracts user-provided credentials (`username` and `password`) from the POST data. Concurrently, the function logs the current timestamp (`moment`) formatted as `%d-%m-%y :%H: %M: %S`. This temporal data, alongside the credentials, is encapsulated within a dictionary (`credentials`) and stored in the session storage, persisting for a duration equivalent to one solar year (365 days).
